AFTER ALL COMMITS ARE DONE — HOW TO ACTUALLY EXECUTE THIS
===================================

STEP 1 — SETUP (everyone does this once)
  git clone https://github.com/<org>/Eventura.git
  cd Eventura
  git fetch origin
  git switch dev
  git pull origin dev

STEP 2 — WORKING ON YOUR MODULE BRANCH (repeat for each commit)
  git switch module/<your-branch>
  # make your changes...
  git add .
  GIT_AUTHOR_DATE="2026-03-01 10:00:00" \
  GIT_COMMITTER_DATE="2026-03-01 10:00:00" \
  git commit -m "your commit message here"
  git push origin module/<your-branch>

STEP 3 — PUSHING TO DEV (after all your commits are done)
  git switch dev
  git pull origin dev
  GIT_AUTHOR_DATE="2026-03-02 11:00:00" \
  GIT_COMMITTER_DATE="2026-03-02 11:00:00" \
  git merge module/<your-branch> --no-ff -m "merge(<branch>): your merge message"
  git push origin dev

STEP 4 — FINAL RELEASE TO MAIN (Harsith does this last)
  git switch main
  git pull origin main
  GIT_AUTHOR_DATE="2026-03-08 18:00:00" \
  GIT_COMMITTER_DATE="2026-03-08 18:00:00" \
  git merge dev --no-ff -m "release: merge dev into main, Eventura v1.0"
  git push origin main
