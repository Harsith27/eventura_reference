HIGH-LEVEL RULES
1. All module branches are created from dev.
2. main receives code only from stabilized dev.
3. Team uses merge commits, not rebase.
4. All contributors pull latest dev before starting feature work each day.
5. Shared file changes must be announced in team chat before merge.
6. Prisma schema changes require explicit coordination.
7. Route-level integration changes require smoke testing before merge.
8. Root cleanup happens near release, not during early migration.
9. Every member keeps commits small, meaningful, and scoped to one logical change.
10. Commit history should reflect actual work done.

REAL-WORK INTEGRITY RULES
1. Each contributor must commit only work they actually performed.
2. Commit author names must match the actual contributor.
3. Commit dates must reflect the actual time of work or a legitimate recorded team workflow.
4. Do not invent fake merge conflicts just for appearance.
5. If conflicts occur naturally, resolve them properly and keep the merge commit.
6. If demo requires history explanation, use true collaboration evidence.
